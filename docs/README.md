# Documentation

AXIOM documentation is split into two layers:

- `docs/` contains human-facing architecture notes, planning material, and roadmap documents.
- `spec/` contains the normative machine-consumable YAML specification that defines the language.

## Structure

### Architecture

- `architecture/architecture-comparison.md` - explains the shift from text-first programming languages to AXIOM's graph-first, agent-native model

### Planning

- `planning/language-creation-plan.md` - phased implementation roadmap for the language and toolchain
- `planning/required-skills.md` - capability map and skill inventory needed across implementation phases

## Recommended Reading Order

1. `../spec/scope.yaml`
2. `../spec/agent-language-design.yaml`
3. `architecture/architecture-comparison.md`
4. `planning/language-creation-plan.md`
5. `planning/required-skills.md`
